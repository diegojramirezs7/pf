Twitter Bot

Python-based Twitter bot that is scheduled to tweets/retweet content
obtained from different websites and APIs at constant periods of time.

Project Description

After discovering the power of APIs in another one of my projects, I decided to build this program that would get content from a variety of sources and tweet things I think are interesting. That way, my Twitter feed would have better content every once in a while. By building this program I would improve my skills in different areas, because there were several problems I had to solve.

  • Scraping tweetable content from a set of websites.
  • Do some web crawling by finding links on seed websites and then scraping the content from the websites that belonged to those links.
  • Get and process content from different APIs.
  • Using the Twitter API to tweet and retweet about a set of predefined topics.
  • Programatically downloading, processing and uploading images.
  • Containerizing the program with Docker to able to run it anywhere.
  • Uploading the containerized program to AWS.

I chose 6 topics that were interesting to me: poetry, comedy, sports, science, technology, and art. To make the tweets/retweets unpredictable, each topic has a list sub-topics and the script randomly selects one of them. There's also a list of content sources for each topic, and the content source for each tweet is also randomly selected. As a final factor for unpredictability, the script can tweet original content with a probability of 50%, tweet news about the selected sub-topic with a probability of 30% or retweet a randomly selected tweet related to a sub-topic with a probability of 20%.

The program is containerized using Docker and the Docker container runs on an AWS EC2 instance. The whole program is written on Python, but several libraries are used.
