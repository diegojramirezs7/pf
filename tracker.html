GPS tracking system for food delivery

This project consists of a food ordering and delivery system where users
can track the status of the order and see the live reduced location
of the delivery driver through their browser.

Project Description

The goal of this project was to build a complete system that would implement a geolocation feature with the help of a Data Stream Management System (DSMS). This feature was to be implemented while preserving the privacy of users by allowing them to choose how much data to share. Our main problem was figuring out how to limit the amount of geolocation data that this kind of systems collect from their users, which can result in privacy concerns.

Given these requirements, our team decided to build a food ordering and delivery system in which users can track the reduced location of the delivery driver. This reduced location would be updated every few seconds rather than instantly and it'd only show a wider area rather than the exact location. The exact location would only be shared when the driver explicitly allows it. The full system consisted of a server implemented with PHP, a MySQL database, the DSMS, and a UI written in React for delivery drivers, customers and the restaurant. The location data was obtained from the browser location API and was displayed using the OpenStreetMap API.

This system was similar to SkipTheDishes or Uber Eats, with the key difference of the reduced location feature. This feature seeked to aliviate the growing concerns of how much data is collected from users, specially geolocation data. By making all data go through the DSMS first and by setting different queries on the DSMS which filter the data sent to the server, we managed to limit the data collected from users.

Development Process

Our team consisted of 7 people working in an agile environment. Sub-groups were created to focus on particular parts of the system such as UI, DSMS and server. However, every person contributed to every part of the program. My worked focused on the implementation of the server, which was the central component of the system. I was responsible for writing the PHP methods that would take HTTP requests from the UIs and the DSMS and responded with the appropriate data, querying the MySQL database if needed.

Unfortunately, the source code for this project was not allowed to be made public due to university policies.
